**Key Findings**:
- **3 npm packages exist** for LM Studio/OpenCode integration: `opencode-lmstudio`, `opencode-local-provider`, `opencode-config-wizard`
- **The `npm` field in provider config** with `@ai-sdk/openai-compatible` is the canonical pattern — OpenCode uses Vercel AI SDK under the hood
- **Auth must be stored separately** in `~/.local/share/opencode/auth.json` (placeholder key for local endpoints)
- **Two-step config**: (1) `opencode.json` defines endpoint + models, (2) `/connect` or direct auth.json edit stores API key
- **No auto-discovery**: You must manually list model IDs matching what your server returns from `GET /v1/models` (feature request #6231)
- **Tool-calling quality**: Depends on local model — Qwen-Coder variants recommended
